vector 应用于视频信息的统计

发表于 2017-06-25 | 分类于 视频处理 ， C++

背景介绍

目前的视频样本总共有 12.3GB 大小，包含 6 种后缀名不同的视频文件，而每一种后缀名视频类型又可以按照分辨率进一步划分为多种类型。因此，为了梳理适合进行分类训练的样本，在此根据不同的指标将现有视频进行分类。

  1. 分辨率（比特率）
    “分辨率”被表示成每一个方向上的像素数量，常见的有 640×480，1024×768，1280×1024 等几种。
  2. 码率
    码流(Data Rate)是指视频文件在单位时间内使用的数据流量，也叫码率或码流率，通俗一点的理解就是取样率，是视频编码中画面质量控制中最重要的部分，一般我们用的单位是kb/s或者Mb/s。
  3. 帧速率
    帧速率也称为FPS(Frames Per Second)的缩写——帧/秒。是指每秒钟刷新的图片的帧数，也可以理解为图形处理器每秒钟能够刷新几次。
  4. 画面宽长比
    4:3(1.33:1) 是历史最久的比例，它在电视机发明之初就已经存在，到现今仍在使用，并且用于许多电脑显示器上。16:9(1.78:1)是高清晰度电视的国际标准，用于澳洲、日本、加拿大和美国，还有欧洲的卫星电视和一些非高清的宽屏幕电视（EDTV）PAL-plus。
  5. 时长
    时长是指一个视频的时间长度。一个好的样本光是满足前 4 条还不够，必须有较长的时长才能提供丰富的台标样本。

从视频文件中提取台标样本

发表于 2017-06-25 | 分类于 视频处理 ， C++

台标提取方法与结果对比

台标样本由视频样本中挑选有限数量的帧产生，为了在单个的视频中尽可能产生互不相同的样本，需要对视频流中相邻前后两个样本做最大相似度限定，使得取后一个样本时，大于相似度限定值的帧被过滤，符合条件的帧成为新的样本。

对于两个帧图像的相似度有不同的量化标准，现有直方图量化法与逐像素取差量化法。前者的计算方法为，计算所有图像的 H-S 直方图，并归一化以便对比。后者的为，统计两张图相似像素的占比。

为了对比两种方法的差异进行了实验。直方图量化法采用 compareHist 函数（参数为 CV_COMP_CORREL），相似度阈值设置为0.05，当为1时表示完全相同。逐像素量化法的占比阈值设置为 70%，当 100% 时几乎完全相同。实验结果如图下图 1 图 2 所示。

OpenCV 实现视频静态台标的提取

发表于 2017-06-13 | 分类于 视频处理

背景介绍

电视台台标在中国出现在电视界面的左上角，标志着电视节目的版权信息。数据统计与监管部门可以通过电视台标统计收视率，监管电视节目的内容。过去的监管需要大量的人力，现在深度学习的出现可以通过台标的识别，减少人力成本。而台标提取是台标识别的前提条件。
根据台标本身是否存在动画，可以分为动态台标与静态台标，本文主要针对静态台标进行提取。而静态台标又可以分为不透明台标与半透明台标。静态台标具有时空不变性，包括位置不变，轮廓不变，颜色不变。利用这种特性，不透明台标可以用基于帧间像素差分的检测方法提取。但是对于透明台标，台标颜色随背景颜色变化，帧间像素差分法不适用。
本文采用了基于边缘检测的提取方法。透明台标虽然颜色会随着背景而变化，但是边缘轮廓会保持不变。另外，台标的识别不仅需要边缘信息，还需要颜色信息等。因此，本文的目标是在边缘算法的基础上提取出完整的台标原图。

操作系统 Win10 64bit，IDE 软件采用 VS 2012，视觉库版本是 OpenCV 2.4.12。

本文采用的方法及流程

  1. Canny 边缘检测并进行时间加权
    Canny 边缘检测算法内置于 OpenCV 中，使用 Canny 边缘检测产生图像的轮廓，白线代表边缘，黑色代表背景。
    为了在视频的多个边缘中筛选出台标的边缘，本文利用台标的时空不变性对 5 帧（每秒取一帧）图像做加权，变动的边缘灰度被减弱，固定的台标边缘灰度保持不变。
    最后经过阈值操作分离出相对干净的台标边缘。
  2. 分水岭操作提取边缘内的台标原图
    分水岭操作的目的是利用边缘图像，生成覆盖台标的 mask，利用 mask 提取原图像帧中的台标。
    分水岭函数需要原图与标识图，而标识图可以由前景与背景合成。因此，本文通过闭运算与膨胀操作生成前景，通过对前景的再膨胀与阈值反色生成背景。
    最后将原图与 mask 做乘法，提取出视频中的台标原图。

Word 的分节符与页眉页脚使用

发表于 2017-05-17 | 分类于 工具

版本说明

Office Word 2016

分节符的分类

分隔符分为分页符与分节符。页是节的组成单元，所以在调整章节之间的关系时，更多的是留意分节符。
分节符分为：

  • 下一页
    在当前页插入分节符，则从下一页开始就是新的一节。
  • 连续
    当前页的分节符后面就是新的一节。使用这种类型的分节符的最常见的原因之一是，如果您正在使用的列。使用此分节符可以更改列的数，而不开始一个新页。
  • 奇数页
    在当前页插入分节符，则从下一个奇数页开始就是新的一节。
    如果当前页是偶数页，则该“奇数页”分节符效果与“下一页”相同；
    如果当前页是奇数页，则下一页在打印成 PDF 时成为空白页（无页眉页脚），但在 Word 软件界面空白页不会显示。从下一个奇数页（也就是“下下页”）开始新的一节。
  • 偶数页
    与奇数页同理，只不过奇数变成了偶数。

不同节的页眉页脚的耦合与去耦合

典型的去耦合场景有，每一节都需要从新的页码“1”开始，首页不显示页码，或者每一节的页眉文字内容不相同。
“耦合/去耦合”的选项出现在双击页眉或页脚的编辑模式下，选项名称为“链接到上一条页眉”。如果选项被选中，则意味着页眉页脚为常量时内容与上一节的一致，为变量时内容默认延续上一节的变量（如页码）。如果选项没有被选中，则页眉页脚在新的一节里重新自定义。

注意事项

不管什么分节场合，强烈不建议在页眉页脚编辑模式下点击“首页不同”。“首页不同”是分节逻辑的快捷按钮，容易造成不分青红皂白地统一或清除页眉页脚格式。因此，即便是需要达成“首页不同”的效果，也从去耦合选项中实现，分节的逻辑更清晰些。
其次，Word 的默认设置是“链接到上一条页眉”，因此每次在新的节中设置页眉页脚，都需要提前注意是否需要去耦合，否则造成后续的编辑混乱。

虚拟机中 Ubuntu16.04 系统 VMTools 的安装

发表于 2017-05-14 | 分类于 工具

遇到的问题

在虚拟机 Ubuntu 系统界面，点击选项安装 VMTools 工具时，出现如下对话框：

正在进行简易安装时，无法手动启动 VMware Tools 安装。

因此，在虚拟机设置选项中选择“软盘选项”，将原来的“使用软盘映像文件”改为“使用物理驱动器”，并加载 VMWare 安装根目录下的 linux.iso 镜像文件，发现软盘里的内容并不能直接出现在桌面，而是在 floppy0 disk 中。
然而，即便以 root 管理者身份登录系统也无法将 floppy0 disk 的压缩包文件到桌面，出现错误提示为：

Error splicing file: Input/output error

软硬件信息

操作系统：i386 32bit 处理器，win7 32bit
使用软件:VMWare Workstation10
虚拟机镜像:Ubuntu 16.04

解决方法

不从软盘上加载镜像，转为直接从一个光盘中“使用物理驱动器”加载，而把剩余的光盘与软盘设置为自动检测。

结果是系统能直接将镜像的内容读取到 Ubuntu 的桌面，不需要从外设中复制。使得剩余的步骤快捷方便。
