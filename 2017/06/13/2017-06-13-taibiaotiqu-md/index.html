OpenCV 实现视频静态台标的提取

发表于 2017-06-13 | 分类于 视频处理

背景介绍

电视台台标在中国出现在电视界面的左上角，标志着电视节目的版权信息。数据统计与监管部门可以通过电视台标统计收视率，监管电视节目的内容。过去的监管需要大量的人力，现在深度学习的出现可以通过台标的识别，减少人力成本。而台标提取是台标识别的前提条件。
根据台标本身是否存在动画，可以分为动态台标与静态台标，本文主要针对静态台标进行提取。而静态台标又可以分为不透明台标与半透明台标。静态台标具有时空不变性，包括位置不变，轮廓不变，颜色不变。利用这种特性，不透明台标可以用基于帧间像素差分的检测方法提取。但是对于透明台标，台标颜色随背景颜色变化，帧间像素差分法不适用。
本文采用了基于边缘检测的提取方法。透明台标虽然颜色会随着背景而变化，但是边缘轮廓会保持不变。另外，台标的识别不仅需要边缘信息，还需要颜色信息等。因此，本文的目标是在边缘算法的基础上提取出完整的台标原图。

操作系统 Win10 64bit，IDE 软件采用 VS 2012，视觉库版本是 OpenCV 2.4.12。

本文采用的方法及流程

  1. Canny 边缘检测并进行时间加权
    Canny 边缘检测算法内置于 OpenCV 中，使用 Canny 边缘检测产生图像的轮廓，白线代表边缘，黑色代表背景。
    为了在视频的多个边缘中筛选出台标的边缘，本文利用台标的时空不变性对 5 帧（每秒取一帧）图像做加权，变动的边缘灰度被减弱，固定的台标边缘灰度保持不变。
    最后经过阈值操作分离出相对干净的台标边缘。
  2. 分水岭操作提取边缘内的台标原图
    分水岭操作的目的是利用边缘图像，生成覆盖台标的 mask，利用 mask 提取原图像帧中的台标。
    分水岭函数需要原图与标识图，而标识图可以由前景与背景合成。因此，本文通过闭运算与膨胀操作生成前景，通过对前景的再膨胀与阈值反色生成背景。
    最后将原图与 mask 做乘法，提取出视频中的台标原图。

1. Canny 边缘检测并进行时间加权

首先获取同目录下的视频文件，实例化 VideoCapture。实例化的参数写成 “VideoSample.mp4” 会出现报错，在这里要加上 “../“。
get 方法可以获取视频每秒的帧数与画面大小信息。

此处为后续的加权准备一个帧作为初始值。
set 方法设置读取帧的位置，此处以毫秒(ms)为单位设置。接着截取感兴趣的 ROI 区域。
canny 函数的高低阈值设为 200 与 100，输出到 frame_ROI 本身。紧接着进行阈值二值化，使图像非黑即白。


每一次循环都使 count 增 1，同时调整 alpha 与 beta 的系数，利用 addWeighted 方法对当前帧与累加帧进行加权。count 取值为 5 是考虑到对视频的取样时间不能太长，否则对于换台等界面切换会出现误判。

2. 分水岭操作提取边缘内的台标原图



此处代码是获取分水岭操作所需要的前景与背景。Morphology 和 Dilation 是对形态学变换中闭运算与膨胀的简单封装。取背景的膨胀操作用了更大的内核尺寸(size)，膨胀效果更强。如果内核尺寸较小，背景的范围无法覆盖图标大小，图形 frame_bg 中间出现的穿孔会导致后续分水岭效果打折扣。因此，取背景的内核大小尽可能大些也不要小。

取背景后又一次使用了二值化阈值函数（THRESH_BINARY_INV 而不是 THRESH_BINARY），灰度大于 1 的像素都设置为 0，否则为 128（灰色）。
将前景与背景相加获得标识图 markers1。

WatershedSegmenter 是对分水岭函数封装的一个类，其中定义了方法：设置标识图 setMarkers，分水岭处理 process，返回分割图像 getSegmentation，返回分割线 getWatersheds。

segmenter1 是这个类的实例化，依次调用方法 setMarkers、process、getSegmentation，则可以获取分割图像 frame_maskimage。为了进行掩码操作，再次调用了阈值函数，使大于 240 的灰度值都为 1，最后与原图做乘法。更加能感受到图像处理就是矩阵的运算。
